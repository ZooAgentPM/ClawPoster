# 小红书封面文案方法论

## 诊断框架

写标题前先完成两个判断，交叉得出结构选择。

**第一维：内容形态**

| 形态 | 判断标准 |
|------|---------|
| 指南型 | 有步骤、能照做 |
| 观点型 | 一个反直觉的主张 |
| 发现型 | 我找到了一个好东西 |
| 情感型 | 一个打动人的瞬间 |

**第二维：受众点击动机**

| 动机 | 判断标准 |
|------|---------|
| 实用 | 我能用到 |
| 身份焦虑 | 我想重新定义自己的处境 |
| FOMO | 有什么我还不知道的事发生了 |
| 情感共鸣 | 这说的是我 |

两个维度交叉 → 选对应结构。

---

## 结构库

### 结构 A：身份命名
```
[文化原型赢家] + [读者的处境] + 大法/指南
```
适用：观点型 × 身份焦虑
条件：必须借用读者已经相信会赢的文化叙事，不能自造
例：邪修找工作大法、野生程序员上位记

**决策点**：受众有没有"我走错路了"的焦虑？有 → 用这个结构给他一个新身份。

---

### 结构 B：悬念缺口
```
[某件事]最危险/最重要的，不是[显而易见的答案]
[大家以为的答案]，不是这个
```
适用：观点型 × 实用/身份焦虑
条件：缺口后面必须有真正意外的答案，不能是废话
例：年轻时最危险的，不是失败

**决策点**：内容有没有一个反直觉的核心主张？有 → 直接说出来，不要包装。

---

### 结构 C：具体对比
```
[错的做法，结果坏]。[对的做法，结果好]。
```
适用：指南型 × 实用
条件：两侧都必须具体到可视化，不能用抽象词
例：夸他好看，他忘了你。夸他睫毛长，他记住你了。

**决策点**：内容有没有"大家都在做的错误" vs "正确做法"的对比？有 → 用具体细节，不用大词。

---

### 结构 D：情感记忆唤醒
```
[受众的共同情感记忆] + [新事物]
```
适用：发现型 × FOMO
条件：受众必须真实经历过那个记忆，借用它的情感重量
例：今年，我找到了有 Manus 感的产品

**决策点**：目标受众有没有某个共同的强烈情感记忆可以借用？有 → 先唤醒它，再引出新事物。

---

### 结构 E：指南承诺
```
[动作][对象] + 指南/大法 + (实操版)
```
适用：指南型 × 实用
条件：语言必须是受众内部说话的方式，不是创作者的视角
例：撩汉子指南（实操版）

**决策点**：内容是不是可以直接照做的步骤？是 → 用受众自己的语言，加"实操版"建立信任。

## 核心规则

**找核心词**
用户 FOMO 情绪的触发词就是核心词。它先出现，其余所有词的唯一任务是放大它的注意力。

**信息顺序：已知先，未知后**
从读者不认识的东西出发，他没有理由停下来。
- ✅ 你的简历，正在被 .claude/ 取代（简历=已知焦虑，.claude/=新概念）
- ❌ .claude/ 正在取代你的简历（从陌生概念出发，钩子失效）

**一个标题只做一件事**
给身份、或制造缺口、或承诺收益，三者选一，不能叠加。

**具体 > 抽象**
睫毛长 > 好看，邪修 > 非传统路径，2015年 > 过去，Manus > 某款 AI 产品

## 常见错误

| ❌ 错误 | ✅ 正确 | 原因 |
|--------|--------|------|
| 挫折并不可怕 | 年轻时最危险的，不是失败 | 「X并不可怕」句式烂大街，读者看到就划走 |
| 你在调 .claude/ 的时候，其实在写简历 | 你的简历，正在被 .claude/ 取代 | 核心词（简历）被埋在句尾，钩子失效 |
| 这样撩男人，简直要他命 | 撩汉子指南（实操版） | 「男人」是创作者视角，「汉子」是受众内部语言 |
| AI领域最值得关注的新产品 | 今年，我找到了有Manus感的产品 | 泛化描述无法触发FOMO，具体情感记忆才有重量 |
| 成功/创新/努力/品质 | 用具体动词和场景 | 通货膨胀词，读者免疫 |

## 验收检查

写完标题后逐项确认：

1. 核心词（FOMO触发词）在不在前面？
2. 有没有用大词替换了本来可以具体的表达？
3. 是从读者视角出发，还是创作者视角出发？
4. 读者看完第一句，脑子里会产生一个问题吗？
5. 这个标题只做了一件事吗？
